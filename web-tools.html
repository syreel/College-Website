Website Tools - Unit 28 M2 and Unit 28 D2

Notepad ++

For some of the web development I used a program called notepad ++ because it can highlight the syntax that you are using and allows you to read the code much easier. Another benefit of notepad ++ is that it doesn’t add any formatting unlike Microsoft word which puts formatting points at the end of lines for example which can break the code and prevent the website from working properly.

Sublime Text

For the bulk of the programming for the website I used a program called sublime. This is because it also does the syntax highlighting which allows you to read the code much easier especially when the website is long. Another reason why I used sublime is because it can auto-complete small pieces of the code as you are typing it which allows you to focus less on closing individual tags and allows you to focus on the content and making the website look and function better.

Chrome Developer Tools

During the development of my site i used the chrome browser to view my webpage. I used chrome because it is one of the most up to date browsers and supports a high percentage of the css and html specification. Within the chrome browser I used the developer menu to change the CSS on the fly to make sure that the changes I were making definitely did what I wanted them to do. This is also good because it means that if something that I change breaks the page then I can just refresh the page to try again from the saved files. Using this technique it is also much faster to develop the css for the site because you can see the result of the changes that you are making live on the screen and do not have to wait for the file to save and the page to refresh before you can see the result and make changes accordingly.

GitHub

I used GitHub during the development of the website for this project because it allows me to make a save copy of my work in increments as I change parts of the page. This therefore means that if I accidentally delete a file that was actually important and that I need to get back I can just restore my save to an older copy from GitHub. This also means that if a change that I make breaks the website and I cannot work out how to fix it then I can revert my repository to an older version and try again from the latest commit.

Stencyl

In order to develop the game in this website I used a software package called Stencyl. This software is used to develop simple games very quickly and easily. This package was good for making my game because my game is simple and doesn’t need any complicated 3d game engines and therefore Stencyl is the best package for doing this in the shortest package. Stencyl is free for small flash games and therefore it is ideal for learning to program and make games.

Word

In order to type up all the documentation for this project I used word because it is simple and easy to use for typing up lots of documentation and formatting it to make it visually appealing.

Photoshop

In order to make the graphics for the game and the logo for the website I used the adobe Photoshop software because it allows me to make high detail images relatively quickly.

Techniques

HTML

To develop the main frame for the website I used the html programming language this is because this language is simple and allows you to make a basic template of how the website will be structured and allows you to change the content of the website without having to reformat the whole page.

CSS

To change how the website looks and to make it look better I used the CSS language because it integrates with the html frame very well and allow you to change how the framework of the html page looks when displayed in a supported browser. CSS is simple to learn and allows you to quickly edit how the site looks. Using the chrome developer menu I was able to live edit the CSS of the page and see the result in real time speeding up the development process.

JavaScript

To make the menu system on the website I used JavaScript this is because in order to make the website interactive you need to use JavaScript because this has the ability to change how the webpage looks while the page is still loaded on the user’s browser. Because the page can be edited live it is possible for the webpage to completely change without refreshing the page therefore you can have a menu that only shows up when you hover over it. Dropdown menus are only possible because of JavaScript.

JQuery

JQuery is a library for JavaScript that simplifies and streamlines the process of programming in JavaScript. It is very useful for making menus because it allows you to use selectors which are very useful for deciding what parts of the page you want to change much easier than with raw JavaScript.

Stencyl Events

In Stencyl I used the events system in order to decide what I wanted the actors to do and what they should react to. These events are very versatile and allow you to program your actors very quickly and allows for rapid prototyping if required

Disabilities

Colourblindness

In order to account for people with colourblindness the website could have a system where you can customise the colour scheme used on the website so that if you are viewing the site and you are havin difficulty reading the text on the page then you can go to the page settings and change the colour of the text on the page. this information can then be saved in the browser to make sure that every time that user visits your site they are able to see correctly.

Poor Eyesight

in order to compensate for people with bad eye sight the website could use the same system as mentioned above in order to change the size of the text on the screen, for example if somebody has trouble seing the display up close then you could change the size of the text on the page to allow them to see it better.

Customisation

In order to customise the website you could have the same colour changing system as used for people with coulourblindness however instead of making the colour options have more colour seperation to make them easier to read you can give them total control of what your websites colours looks like so the user can choose their experience on your site.
